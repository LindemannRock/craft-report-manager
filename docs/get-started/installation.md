# Installation & Setup

> [!NOTE]
> Pre-Release: Report Manager is in active development and not yet available on the Craft Plugin Store. Install via Composer for now.

## Composer

Add the package to your project using Composer and the command line.

1. Open your terminal and go to your Craft project:

```bash
cd /path/to/project
```

2. Then tell Composer to require the plugin, and Craft to install it:

```bash title="Composer"
composer require lindemannrock/craft-report-manager && php craft plugin/install report-manager
```

```bash title="DDEV"
ddev composer require lindemannrock/craft-report-manager && ddev craft plugin/install report-manager
```

3. **Optional** — Enable [Logging Library](https://github.com/LindemannRock/craft-logging-library) for log viewing:

> [!NOTE]
> Logging Library is included as a Composer dependency and downloaded automatically. Activate it in Craft to enable log viewing.

```bash title="PHP"
php craft plugin/install logging-library
```

```bash title="DDEV"
ddev craft plugin/install logging-library
```

Or via the Control Panel: **Settings → Plugins → Logging Library → Install**

## Post-Install Setup

After installing, complete these steps to get Report Manager working:

### 1. Choose Where Exports Are Stored

Go to **Report Manager → Settings → Export**. By default, exports are written to the local filesystem at `@storage/report-manager/exports`. To store them in a Craft asset volume instead, select a volume from **Export Storage Volume**.

> [!IMPORTANT]
> Export paths must resolve outside the webroot. Use the `@storage` or `@root` aliases — a path that resolves to the webroot is rejected for security.

See [Configuration](configuration.md) for storage, format, and cleanup options.

### 2. Make Sure the Queue Runs

Exports are generated by Craft's queue, not inline. For scheduled reports and timely export generation, run the queue with a daemon (a `queue/listen` worker or a cron-driven `queue/run`). See [Craft's queue documentation](https://craftcms.com/docs/5.x/system/queue.html).

### 3. Create Your First Report

Go to **Report Manager → Reports → New Report**, pick a data source, choose what to export, and save. See the [Quickstart](quickstart.md) for a step-by-step walkthrough.

### 4. Review Configuration

See [Configuration](configuration.md) for all available settings. Most can be managed from **Report Manager → Settings** without a config file.
